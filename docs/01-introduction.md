# 01 — Introduction

Ce document fait partie d’un projet complet d’architecture AWS de niveau production.  
L’objectif est de démontrer ma capacité à concevoir, automatiser et sécuriser une infrastructure Cloud AWS en utilisant :

- Terraform (Infrastructure as Code)
- GitHub Actions (CI/CD)
- AWS Well-Architected Framework
- Sécurité Zero Trust
- Observabilité CloudWatch

Ce projet est structuré comme un vrai projet d’entreprise, avec :

- Une architecture réseau complète (VPC multi-AZ)
- Un plan de compute scalable (ALB + ASG + Launch Template)
- Un pipeline CI/CD automatisé
- Une sécurité renforcée (IAM, CloudTrail, Config, GuardDuty, KMS)
- Une documentation professionnelle en 10 sections

Ce fichier introduit le contexte général du projet.
